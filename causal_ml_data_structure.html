Overview

The framework produces four tables with distinct roles, plus a master reference table. Three tables operate at paper level (one row per paper) and one is long-format. The design separates scored integers, free-text annotations, audit evidence, and framework metadata — keeping each table analytically clean.

framework
framework.tsv
~90 rows · Long format · Composite PK: dimension + level
The rubric itself. One row per dimension–level pair. Lookup for all level descriptions and labels.
scores
scores.tsv
25 rows · Wide format · PK: paper_id
Ordinal scores — one row per paper, one column per scored dimension. Primary analytic table.
descriptive
descriptive.tsv
25 rows · Wide format · PK: paper_id
Free-text annotations. Never scored or averaged. Supports narrative synthesis only.
evidence
evidence.tsv
~550 rows · Long format · Composite PK: paper_id + dimension
Bridge table — excerpts, page refs, rater notes. Connects all other tables. Central audit trail.
All tables use tab-separated .tsv format to avoid conflicts with commas in free-text fields. All paper-level tables link via paper_id. Framework and evidence link via dimension.

Relationship summary

  • papers 1 ──── 1 scores  —  one-to-one on paper_id
  • papers 1 ──── 1 descriptive  —  one-to-one on paper_id
  • papers 1 ──── ∞ evidence  —  one paper has ~22 evidence rows (one per dimension)
  • framework 1 ──── ∞ evidence  —  one dimension maps to 25 evidence rows (one per paper)
  • framework structural scores  —  indirect link: dimension values in framework match column names in scores. Requires pivot_longer before joining — see R reference.
The framework → scores link is structural, not a standard FK join. Scores is wide (dimensions as columns); framework is long (dimensions as row values). To join: pivot scores to long format first.

Table 4 — evidence evidence.tsv

The audit and bridge table. One row per paper × dimension (~22 rows per paper, ~550 rows total). Stores the text excerpts and page references justifying each assigned score. R — Loading & validation

Load all five tables with explicit column type declarations. The anti_join checks should all return 0 rows before any analysis proceeds.

library(tidyverse)

framework   <- read_tsv("framework.tsv",   col_types = cols(.default = "c", level = "i"))
papers      <- read_tsv("papers.tsv",      col_types = cols(year = "i", sample_size = "i", date_scored = "D"))
scores      <- read_tsv("scores.tsv",      col_types = cols(.default = "i", paper_id = "c"))
descriptive <- read_tsv("descriptive.tsv", col_types = cols(.default = "c"))
evidence    <- read_tsv("evidence.tsv",    col_types = cols(.default = "c", score_assigned = "i"))

# Validate joins — all should return 0 rows
anti_join(scores,      papers,    by = "paper_id")
anti_join(descriptive, papers,    by = "paper_id")
anti_join(evidence,    papers,    by = "paper_id")
anti_join(evidence,    framework, by = "dimension")

# Full analytic dataset (scores + metadata)
dat <- papers |> left_join(scores, by = "paper_id")

R — Key join patterns

The framework–scores join requires a pivot because scores is wide and framework is long.

# Get score + framework description for every paper × dimension
scores_long <- scores |>
  pivot_longer(-paper_id, names_to = "dimension", values_to = "score")

annotated <- scores_long |>
  left_join(framework, by = c("dimension", "score" = "level")) |>
  left_join(select(papers, paper_id, first_author, year), by = "paper_id")

# Score + excerpt + rationale for a specific paper
evidence |>
  filter(paper_id == "Smith_2021") |>
  left_join(framework, by = c("dimension", "score_assigned" = "level")) |>
  select(dimension, score_assigned, description, excerpt, page_ref)

R — Domain score computation

Compute domain subscores as row means within each domain's columns. method_type and sample_splitting are excluded — they are flags, not quality measures.

scores <- scores |>
  mutate(
    domain_study_design     = rowMeans(pick(patient_population, comparators), na.rm = TRUE),

    domain_confounding      = rowMeans(pick(confounders_identified, confounding_adjustment,
                                              balance_diagnostics, unmeasured_confounding), na.rm = TRUE),
    # NA for RCT papers handled automatically by na.rm = TRUE

    domain_effect_mod       = rowMeans(pick(predefined_effect_mod, posthoc_effect_mod), na.rm = TRUE),

    domain_ml_method        = rowMeans(pick(method_clarity, outcome_method_alignment,
                                              hyperparameter_tuning, multi_model_comparison), na.rm = TRUE),
    # method_type (nominal) and sample_splitting (binary) excluded

    domain_causal           = rowMeans(pick(causal_estimand, survival_estimand), na.rm = TRUE),

    domain_treatment_effect = rowMeans(pick(clinical_decision_readiness, uncertainty_quantification,
                                              effect_mod_plausibility), na.rm = TRUE),

    domain_reproducibility  = code_availability
  )

R — Inter-rater reliability

If papers are double-scored, rater_id in papers and a long-format reshape of scores enables ICC or weighted kappa per dimension.

library(irr)

scores_long <- scores |>
  pivot_longer(-paper_id, names_to = "dimension", values_to = "score") |>
  left_join(select(papers, paper_id, rater_id), by = "paper_id")

# Two-column rater matrix for one dimension
icc_input <- scores_long |>
  filter(dimension == "causal_estimand") |>
  pivot_wider(names_from = rater_id, values_from = score) |>
  select(-paper_id, -dimension)

icc(icc_input, model = "twoway", type = "agreement")

Key design decisions

Why four tables and not one?

A single flat table would mix integers, free text, long and wide formats, and metadata. Each table has exactly one unit of observation and one content type, making column type declarations unambiguous and every analysis focused.

Why is evidence a separate table and not columns in scores?

Adding excerpt and page_ref directly to scores would require 22 excerpt columns + 22 page_ref columns = 44 extra wide columns, most of which would be redundant. The long-format evidence table puts each excerpt in its own row and joins cleanly to both scores and framework.

Why TSV and not CSV?

Evidence and descriptive contain free-text fields that routinely include commas (method names, variable lists, quoted excerpts). TSV avoids all quoting complications.

Why NA for confounding columns in RCT papers?

Using 0 would imply worst-possible score and incorrectly penalise RCT papers in aggregate analyses. NA propagates correctly through rowMeans(..., na.rm = TRUE).

Why is score_assigned denormalised in evidence?

To make the evidence table self-contained for auditing — a rater reviewing excerpts sees score + excerpt + rationale in one place without running a join. Maintain consistency by updating evidence when scores changes.

Why is method_type in scores but treated as nominal?

Structural consistency — one row per paper, same wide format. But it must never be summed or averaged. Use it as a stratification variable: compare causal_estimand scores across method_type groups.
